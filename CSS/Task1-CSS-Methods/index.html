Understanding CSS: Three Different Approaches

Internal CSS (Embedded in HTML)

This section demonstrates internal CSS styling using the <style> tag in the <head> section.

Pros: Easy to manage for small projects, all styles in one place

Cons: Cannot be reused across multiple pages, increases HTML file size

External CSS (Linked Stylesheet)

This section uses external CSS from the styles.css file linked at the top of the HTML.

Pros: Reusable across multiple pages, better caching, cleaner HTML, easier maintenance

Cons: Requires additional HTTP request

Inline CSS

This section is styled directly with inline style attributes on the HTML element.

Pros: Applies directly to an element, highest specificity

Cons: Difficult to maintain, poor code organization, cannot be reused, bloats HTML

Best Practice Recommendation

For production applications, use External CSS for the following reasons:

  • Separation of concerns (HTML for content, CSS for styling)
  • Better maintainability across large projects
  • Code reusability
  • Browser caching improves performance
  • Easier for teams to collaborate
