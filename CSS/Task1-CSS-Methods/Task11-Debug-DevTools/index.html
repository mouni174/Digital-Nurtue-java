Chrome DevTools for CSS Debugging

DevTools Shortcut: Press F12 (Windows/Linux) or Cmd+Option+I (Mac)

Chrome DevTools provides powerful features for debugging CSS and inspecting your website in real-time.

1. The Inspector Tool

Shortcut: Ctrl+Shift+C (or Cmd+Shift+C on Mac)

Click on me to inspect!

Right-click on any element and select "Inspect" or use the Inspector tool to examine elements and their CSS.

What You Can Do:

  • Select elements on the page and see their HTML structure
  • View all CSS rules applied to an element
  • See the Box Model visualization
  • Identify which CSS file and line number each rule comes from
  • Toggle CSS rules on/off to see their effect
  • Edit CSS values in real-time to test changes
  • Hover over styles to see where they're defined

2. Box Model Inspector

In the Inspector, scroll down to see the Box Model diagram showing margin, border, padding, and content.

Content area (320x150px)

Try this: Inspect this box and look at the box model. The colors show:

  • Orange = Content
  • Green = Padding
  • Yellow = Border
  • Red/Pink = Margin

3. Live CSS Editing

In DevTools, you can edit CSS values live without modifying your source files.

Try Editing This!

Right-click → Inspect. In the Styles panel, modify the background-color, font-size, or padding values. Changes appear instantly!

Steps to Edit CSS Live:

  1. Inspect the element
  2. Find the CSS rule in the Styles panel
  3. Click on any CSS property value
  4. Type your new value
  5. Press Enter to apply
  6. Use arrow keys to increment/decrement numeric values

4. Device Toolbar (Responsive Testing)

Shortcut: Ctrl+Shift+M (or Cmd+Shift+M on Mac)

The Device Toolbar lets you simulate different devices and screen sizes:

Test These Breakpoints:

  • iPhone SE (375px) Popular mobile device
  • iPad (768px) Tablet size
  • Desktop (1920px) Large screen
  • Custom Size Enter any dimensions

What to Check:

  • Navigation menus stack properly on mobile
  • Images scale and remain visible
  • Text is readable at all sizes
  • Buttons and forms are touch-friendly (min 44px)
  • No horizontal scrolling on mobile
  • Media queries activate at correct breakpoints

5. Network Tab (Check CSS Loading)

The Network tab shows all resources loaded by your page, including CSS files.

Steps to Verify CSS is Loading:

  1. Open DevTools (F12)
  2. Click the "Network" tab
  3. Refresh the page (Ctrl+R or Cmd+R)
  4. Look for your CSS file (e.g., styles.css)
  5. Check the Status column (should be 200 or 304)
  6. Status 404 means the file wasn't found
  7. Check the Size column to see the file size

Understanding Status Codes:

  • 200: File loaded successfully
  • 304: Cached file (not modified)
  • 404: File not found (check the path)
  • 500: Server error

6. Common CSS Issues & How to Debug

❌ Styles Not Applied

Cause: CSS file not linked, wrong selector, or typo

Debug:

  • Check Network tab - is CSS file loading?
  • Inspect element - do you see the CSS rule?
  • Check selector specificity - higher specificity wins
  • Look for !important rules that might override
